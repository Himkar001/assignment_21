Q1 — What is NumPy broadcasting? Why is it useful?

Answer: NumPy broadcasting is a mechanism that allows arrays of
different shapes to be used together in arithmetic operations. Instead
of manually reshaping arrays, NumPy automatically expands the smaller
array to match the larger one.

Why it is useful: - Eliminates the need for loops - Makes code shorter
and cleaner - Improves performance by using optimized operations

Example: [1 2 3] + [[10 20 30], [40 50 60]]

Result: [[11 22 33], [41 52 63]]

Q2 — Coding: normalize(X)

Code: import numpy as np

def normalize(X): return (X - np.min(X)) / (np.max(X) - np.min(X))

X = np.array([10, 20, 30, 40]) print(normalize(X))

Output: [0. 0.33333333 0.66666667 1. ]

Explanation: The formula scales all values between 0 and 1 using min-max
normalization.

Q3 — Difference between Vectorisation and Loops

Answer:

Vectorisation: - Applies operations to entire arrays at once - Uses
NumPy functions - Faster and more efficient

Loops: - Processes elements one by one - Slower due to Python overhead

Why NumPy is faster: - Written in C (low-level optimized code) - Uses
vectorised operations - Avoids Python loops - Better memory management

Conclusion: Vectorisation is preferred over loops for performance and
readability.
